# 매일 실행되는 리포트 생성 지시문 (Daily Prompt)

> 이 파일은 예약 작업이 매일 Claude Code에 전달하는 "명령"입니다.
> 스케줄러/루틴은 아래 내용을 그대로 실행하게 됩니다. (설정 방법: AUTOMATION.md)

---

당신은 이 저장소의 주식시장 리포트 자동 생성기다. 오늘 날짜의 시장 리포트를 만들어 게시하라. 순서:

## 1. 데이터 수집 (웹검색)
다음을 웹검색으로 최신 상태 조사한다 (여러 소스 교차 확인):
- 미국: S&P500 · 나스닥 · 다우 최근 종가와 등락, VIX, 미 10년물 금리, 달러
- 한국: 코스피 · 코스닥 지수, 원/달러 환율
- 오늘~이번 주 예정된 주요 이벤트 (실적 발표, 경제지표, FOMC 등)
- 밤사이·오전 시장을 움직인 핵심 뉴스 (지정학·기업·정책)

## 2. 분석 (핵심)
- 단순 "오른다/내린다"가 아니라 **상승 / 보합 / 하락 시나리오에 각각 확률(%)** 을 부여하고 근거를 붙인다.
- 각 시나리오의 **트리거(무엇이 일어나면 그 방향)** 와 **무효화 조건**을 명시한다.
- 한국 시장은 반도체 쏠림 등 구조적 특성을 고려한다.

## 3. 전날 예상 검증
- `_posts/` 폴더에서 가장 최근 리포트를 읽어, 그 예상이 실제로 맞았는지 O/X와 한 줄 코멘트로 평가한다.
- (전날 파일이 없으면 "첫 리포트"로 표기)

## 4. 리포트 파일 생성
- 경로: `_posts/YYYY-MM-DD-market-report.md` (오늘 날짜)
- 맨 위에 Jekyll front matter 포함:
  ```
  ---
  layout: post
  title: "시장 리포트 — YYYY-MM-DD (요일)"
  date: YYYY-MM-DD 08:00:00 +0900
  categories: report
  ---
  ```
- 본문 목차: 0.한줄결론 → 1.글로벌매크로 → 2.이번주이벤트 → 3.시나리오(확률표) → 4.한국코멘트 → 5.리스크/무효화 → 6.어제예상vs실제
- 맨 위에 "투자 조언 아님" 경고 문구를 넣는다.
- 확률 시나리오는 표로 만든다.

## 5. 게시 (git)
- 생성한 파일을 `git add` → `git commit -m "report: YYYY-MM-DD"` → `git push` 한다.
- push가 되면 GitHub Pages 사이트에 자동 반영된다.

## 규칙
- 근거 없는 단정 금지. 모든 판단에 데이터/뉴스 근거를 붙인다.
- 겸손하게, 확률로 말한다. 맞히는 게 아니라 기록·검증이 목적이다.
