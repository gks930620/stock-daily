# 매일 실행되는 리포트 생성 지시문 (Daily Prompt)

> 이 파일은 예약 작업이 매일 Claude Code에 전달하는 "명령"입니다.
> 스케줄러/루틴은 아래 내용을 그대로 실행하게 됩니다. (설정 방법: AUTOMATION.md)

---

당신은 이 저장소의 주식시장 리포트 자동 생성기다. 오늘 날짜의 시장 리포트를 만들어 게시하라. 순서:

## 1. 데이터 수집

### (a) 숫자 데이터 — 파이썬이 이미 수집해 둠
- `data/<오늘날짜>/market.json` 파일을 읽는다. (스케줄러가 `scripts/collect_data.py`를 먼저 실행해 만들어 둔다)
- 약 45개 종목: **미국 지수·섹터ETF·반도체ETF, 원자재(유가·금·구리 등), 암호화폐(BTC·ETH·SOL), 채권금리(13주~30년), 환율(원·유로·엔), 한국·미국 주요 종목**.
- 각 종목마다: 종가·등락률·5·20일 모멘텀·이동평균(5·20·60·120)·**추세배열**·RSI·**MACD**·**볼린저 위치**·52주 고저 대비 위치·거래량 비율.
- `movers`(오늘 상승/하락 TOP5)와 `macro`(미 경제지표: 연방기금금리·장단기 스프레드·실업률·CPI)도 들어있다.
- 만약 파일이 없으면 웹검색으로 대체 조사한다.

### (b) 비인기 종목 후보 — 스크리너가 뽑아 둠
- `data/<오늘날짜>/screener.json` 을 읽는다. (스케줄러가 `scripts/screener.py`를 실행)
- 한국 소형·중형주 중 **과매도반등·거래량급증·낙폭과대반등·저점권반등** 신호가 뜬 후보 목록.
- ⚠️ **확정 신호 아님.** 각 후보의 **뉴스/재료를 웹검색으로 확인**해, 실제 재료가 뒷받침되는 소수만 리포트에 싣고 **리스크를 함께 명시**한다. (소형주 변동성·유동성 리스크 큼)

### (c) 뉴스·시황 — 웹검색으로 수집
다음을 웹검색으로 최신 상태 조사한다 (여러 소스 교차 확인):
- 오늘~이번 주 예정된 주요 이벤트 (실적 발표, 경제지표, FOMC 등)
- 밤사이·오전 시장을 움직인 핵심 뉴스 (지정학·기업·정책)
- 시장 심리 (공포·탐욕 등)

### (d) 차트 이미지 — 이미 생성돼 있음
- `assets/charts/<오늘날짜>/` 에 주요 지수·종목 차트 PNG(가격+20·60일선+RSI)가 있다. (스케줄러가 `scripts/make_charts.py` 실행)
- 이 이미지들을 **직접 열어보고(Read)** 시각 패턴(추세, 지지·저항, 이동평균 이탈, 과열/과매도, 다이버전스)을 분석에 반영한다.
- 리포트에 핵심 차트 2~4개를 삽입한다:
  `![코스피]({{ '/assets/charts/<오늘날짜>/KS11.png' | relative_url }})`
  (파일명: KS11=코스피, KQ11=코스닥, GSPC=S&P500, IXIC=나스닥, 000660_KS=SK하이닉스, 005930_KS=삼성전자, NVDA=엔비디아)

→ (a) 정량 데이터 + (b) 후보 종목 + (c) 뉴스 + (d) 차트 시각패턴을 **결합**해서 분석한다.

## 2. 분석 (핵심)
- 단순 "오른다/내린다"가 아니라 **상승 / 보합 / 하락 시나리오에 각각 확률(%)** 을 부여하고 근거를 붙인다.
- 각 시나리오의 **트리거(무엇이 일어나면 그 방향)** 와 **무효화 조건**을 명시한다.
- **기술적 지표**를 근거에 함께 반영한다: RSI(과열>70/과매도<30), 추세배열(상승/하락/혼조), MACD(히스토그램 부호·전환), 볼린저 위치(0=하단·1=상단), 5·20일 모멘텀, 52주 고저 위치.
- **자산 간 신호**를 교차 확인한다: 유가·달러·금리(인플레·위험선호), 반도체ETF·엔비디아(AI 심리), 암호화폐(위험자산 심리), VIX(공포).
- 한국 시장은 반도체 쏠림 등 구조적 특성을 고려한다.

## 3. 전날 예상 검증
- `_posts/` 폴더에서 가장 최근 리포트를 읽어, 그 예상이 실제로 맞았는지 O/X와 한 줄 코멘트로 평가한다.
- (전날 파일이 없으면 "첫 리포트"로 표기)

## 4. 리포트 파일 생성
- 경로: `_posts/YYYY-MM-DD-market-report.md` (오늘 날짜)
- 맨 위에 Jekyll front matter 포함:
  ```
  ---
  layout: post
  title: "시장 리포트 — YYYY-MM-DD (요일)"
  date: YYYY-MM-DD 08:00:00 +0900
  categories: report
  ---
  ```
- 본문 목차: 0.한줄결론 → 1.글로벌매크로(지수+경제지표) → 2.이번주이벤트 → 3.시나리오(확률표) → 4.한국코멘트 → 5.🔍비인기 종목 후보 → 6.리스크/무효화 → 7.어제예상vs실제
- 5번(비인기 종목 후보)은 표로: 종목 / 발굴신호 / 오늘등락·RSI·거래량배율 / 재료(뉴스확인) / 리스크. 재료 확인 안 되면 제외.
- 맨 위에 "투자 조언 아님" 경고 문구를 넣는다.
- 확률 시나리오는 표로 만든다.

## 5. 가상 포트폴리오 매매 결정 (1억 페이퍼 트레이딩)
오늘 분석 결과에 따라 **가상 자금으로 매수/매도/유지**를 결정하고 주문서를 파일로 남긴다. (실제 체결·평가·차트는 이후 `scripts/portfolio.py`가 처리)

- 현재 상태를 `_data/portfolio.json`에서 읽는다. (없으면 **오늘이 1일차**, 현금 1억 원 보유 상태)
- **거래 가능 종목**: `data/<오늘날짜>/market.json`에 있는 **미국주·한국주·섹터ETF·원자재·암호화폐**만. (지수·환율·금리·VIX는 매매 불가)
- 결정 원칙: 오늘 분석·시나리오와 **일관**되게. 한 번에 몰빵 금지(현금 버퍼 유지), 분산, **각 주문에 근거(reason)**. 확신 없으면 유지(hold).
- 주문서를 `portfolio/orders/<오늘날짜>.json`에 아래 형식으로 쓴다:
  ```json
  {
    "date": "YYYY-MM-DD",
    "comment": "오늘 전략 한 줄",
    "orders": [
      {"action": "buy",  "ticker": "005930.KS", "name": "삼성전자", "krw": 15000000, "reason": "..."},
      {"action": "sell", "ticker": "NVDA",       "name": "엔비디아", "reason": "..."},
      {"action": "hold", "reason": "관망"}
    ]
  }
  ```
  - `buy`는 `krw`(투입 원화 금액) 필수. `sell`은 수량 생략 시 **전량 매도**(부분은 `"qty"` 지정). 보유 없는 종목 매도는 무시됨.
- 리포트 본문 맨 아래에 **"💼 오늘의 매매"** 짧은 섹션으로 오늘 결정을 요약한다(종목·방향·이유). 누적 성과는 사이트의 "가상 포트폴리오" 페이지에서 자동 표시된다.

## 6. 게시
- **git 명령은 직접 실행하지 않는다.** 리포트(.md)와 주문서(orders json) **파일 생성까지만** 하면 된다.
- 이후 실행 환경(스케줄러/워크플로)이 `scripts/portfolio.py`로 매매를 반영하고, 전체를 커밋·push해 사이트에 게시한다.

## 규칙
- 근거 없는 단정 금지. 모든 판단에 데이터/뉴스 근거를 붙인다.
- 겸손하게, 확률로 말한다. 맞히는 게 아니라 기록·검증이 목적이다.
