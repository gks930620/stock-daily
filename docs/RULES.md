# 운영 규칙서 (실수 방지 기준)

> 매일 자동 실행(예상글·매매·게시)이 따라야 하는 확정 규칙. 프롬프트(`prompts/kr-report.md`·`us-report.md`·`portfolio.md`)와
> 코드(`scripts/`)에 이미 반영되어 있으며, 이 문서는 **왜 그렇게 정했는지**와 **재발 방지 체크리스트**의 원본이다.

---

## 0. 역할 분리 — AI 4명 (2026-07-14 확정)

한 회차에 **서로 다른 Claude 세션(=AI 4명)** 이 순서대로 일한다. 헤드리스 `claude -p`를 네 번 호출 → 세션이 분리되어 기억을 공유하지 않는다. **먼저 성향이 다른 매니저 3명(①②③)이 각자 종목을 확정**하고, **마지막에 애널리스트(④)가 그 3인의 선택을 종합해 "오늘의 매수/매도 종목"을 정리**한다. (2026-07-15 순서 변경: 리포트가 3인 뒤로 이동.)

| 순서 | 담당 | 프롬프트 | effort | 하는 일 | 산출물 |
|---|---|---|---|---|---|
| ① | 🛡️ **안정형** 매니저 | `persona-stable.md` + `portfolio.md` | high | 데이터 직접 분석 → 종목 매수/매도 확정 | `portfolio/orders/<날짜>-<세션>-stable.json` |
| ② | 🚀 **공격형** 매니저 | `persona-aggressive.md` + `portfolio.md` | high | 성장·모멘텀 관점 매매 | `…-aggressive.json` |
| ③ | 🎯 **역발상형** 매니저 | `persona-contrarian.md` + `portfolio.md` | high | 컨트래리안 관점 매매 | `…-contrarian.json` |
| ④ | 🧠 **애널리스트** | `<kr\|us>-report.md` | xhigh | **3인 주문서 3개를 종합** → 종목별 매수/매도 결론 리포트 | `_posts/<날짜>-<kr\|us>-market.md` |

- 3명은 **같은 `market.json`을 보되 성향대로 다르게** 매매(리포트는 아직 없음 — 각자 1차 결정) → 각자 독립 계좌 `_data/portfolio-<id>.json`. 목적: **어느 성향이 실제로 버는지** 정직하게 비교.
- 애널리스트는 3인의 주문서(`orders/<날짜>-<세션>-*.json`)를 읽고 **두루뭉술한 시황이 아니라 "이 종목 사라/팔아라"** 를 종목 단위로 종합한다(편집 결론 — 실행은 안 함).
- 데이터 수집·차트·스크리너·체결계산은 **파이썬**(`scripts/portfolio.py <label> <persona>`, 성향별 3회)이 한다.
- **손익 비교 가격의 출처는 파이썬이 수집한 `data/<날짜>/market.json` 하나** — 세 계좌의 체결·평가가 모두 이 동일 파일 기준. 그래서 세 성향이 같은 잣대로 채점된다.
- 경계: 애널리스트는 주문/계좌를 안 건드린다. 매니저는 글(`_posts`)·남의 계좌를 안 건드리고 자기 주문서 하나만 만든다. 모두 git 금지(커밋은 워크플로).
- 비용: 회차당 세션 **4개**(high 3 + xhigh 1) → 토큰 소모 큼. Max 리밋이 부담되면 매니저 effort를 `medium`으로 낮추거나 성향 수를 줄이는 선택지(`daily.yml`).
- 페이지: `/portfolio/`=3인 비교 허브, `/portfolio/<id>/`=성향별 상세(증권앱형).

## 1. 스케줄 (하루 2회 · 장 마감 직후 · 평일 KST)

> 2026-07-15 개편: **개장 전(08시)이 아니라 "장 마감 후"에 실행**한다. 이유 — 오를 종목은 다음날 개장(09시)에 이미 갭으로 다 반영돼 개장가엔 못 잡는다. **종가에 담으면 밤사이 갭을 미리 먹는다**(내가 이미 보유). 종가는 그 시간에 실제 거래 가능(🇰🇷 15:40~16:00 장후 시간외 종가).

| 시각 (KST) | 모드 | 하는 일 |
|---|---|---|
| **16:00** (07:00 UTC) | `kr` | 🇰🇷 한국장 마감(15:30) 후 — 수집+차트+스크리너 → 매매결정 3인 → 종합 리포트 → **종가 체결·평가** |
| **06:30** (21:30 UTC) | `us` | 🇺🇸 미국장 마감 후 — 수집+차트 → 매매결정 3인 → 종합 리포트 → **종가 체결·평가** |

- 우리는 **"이 종목 사라/팔아라"를 종목 단위로 콜**한다. 종가에 담아 **"내일 이후에도 갈 종목"** 을 노린다(오늘 이미 오른 것 추격 아님).
- 검증 루프: 어제 종가 추천 종목이 **오늘 종가까지** 올랐나/내렸나(종가→종가) 대조(O/△/X).

## 2. 글 포맷 — PPT 슬라이드 + 접이식 전체 글 (2026-07-15 확정)

> 리포트는 3인 뒤에 실행되어 주문을 종합한다. **결론만 한눈에 보이는 PPT 슬라이드 5장**이 본문이고, 자세한 근거는 접이식(`<details>`)으로 숨긴다 — "PPT랑 리포트 둘 다".

- front matter에 **`slides: true` 필수** → `post` 레이아웃이 전체폭 슬라이드 모드로 렌더(`_layouts/post.html` + `_includes/ppt-style.html`).
- 본문 = `<div class="ppt">` 슬라이드 5장 + `<details class="full-report">` 상세.
  1. **01 표지·결론** — `.cover`: 큰 제목 + `🔴 매수 (종목) / 🔵 회피 (종목)` (매수=빨강 `.call.buy`, 회피=파랑 `.call.sell`)
  2. **02 시장 한눈에** — `.grid`의 `.stat` 타일 4~6개(상승 `.up`▲빨강 / 하락 `.down`▼파랑)
  3. **03 3인의 선택** — `.three`의 `.who` 3칸(성향·매수종목·현금%)
  4. **04 종합 매수/매도** — `.verdict-list`의 `.vl.buy`/`.vl.sell` (종목 + 근거 + `.agree` 3인중 누구) — **핵심**
  5. **05 핵심 한 줄** — `.big .huge` 대립 두 단어 + 포트폴리오 링크
  - `<details class="full-report">` — 종목별 상세 근거 · 3인 상세 · 어제 채점(마크다운 서술)
- 슬라이드 CSS 소스: `_includes/ppt-style.html` (예시 페이지 `/report-ppt/`도 동일 사용).
- 색 규칙(한국식): **매수·강세=빨강**, **매도·약세=파랑**. 파일명 `_posts/YYYY-MM-DD-<kr|us>-market.md`.
- front matter 필수: `market: kr|us` (목록이 🇰🇷 한국장 / 🇺🇸 미국장 **두 갈래로 분리**됨) · `slides: true` · `date: YYYY-MM-DD HH:MM:00 +0900` — **HH:MM은 실제 작성 시각**(워크플로가 `[실행 안내]`로 넘겨줌). 목록·글 상단에 "작성 시각"으로 분까지 표시된다.

## 3. 가상 포트폴리오 체결 규칙 (1억 페이퍼 트레이딩)

**매매는 하루 단위** (실시간 아님). AI가 **장 마감 후** 분석해 **당일 종가로 체결**한다. 진입가(종가)는 확정됐지만 손익은 **이후 종가**로 결정 → **룩어헤드(결과 보고 사기) 불가능**.

| 규칙 | 내용 | 이유 |
|---|---|---|
| 체결가 | **모든 주문 = 당일 종가로 즉시 체결** (🇰🇷 15:30 마감 후·🇺🇸 마감 후 실행). 종가는 그 시간에 실제 거래 가능(🇰🇷 15:40~16:00 장후 시간외 종가) | 개장 갭을 놓치지 않으려 종가에 미리 진입 |
| 룩어헤드 방지 | 손익은 아직 안 나온 **다음 날 이후 종가**로 결정. 종가에 미리 들어가므로 밤사이 갭이 **유리**하게 작동 | "오늘 이미 오른 것"이 아니라 "내일 갈 것"을 고르게 함 |
| 평가 | **장 마감 종가 기준**, 매 실행마다 일별 스냅샷 확정 → 일별현황·자산곡선 | 매 마감 같은 방식으로 공정 채점 |
| 주식·ETF | **정수 주수만** 체결(한국·미국 공통), 잔액은 현금 유지 | 64.3주 같은 비현실 방지 |
| 미국 자산 | **달러 가격으로 평단·현재가 표시** + 체결 시점 환율(원/달러) 기록 | 원화 환산만 보면 왜곡 |
| 암호화폐·원자재 | 24시간 자산 → 소수 단위 체결(단위: oz/BTC 등, "주" 아님) | 0.5 BTC·1.9 oz는 현실적 |
| 주문서 | `portfolio/orders/<날짜>-<kr|us>.json` — buy는 krw(예산), sell은 qty 생략 시 전량 | 세션별 1회 적용(`applied_orders`) |
| 원칙 | 몰빵 금지·분산·각 주문에 근거(reason)·확신 없으면 hold | 이벤트 직전(발표 전) 추가 베팅 금지 |
| 검증 | 모든 수치는 커밋된 `data/`·`orders/`·`_data/portfolio.json`과 대조 가능 | 신뢰성 — "대충 가상" 금지 |
| 미반영 | 수수료·세금·슬리피지 없음 (명시할 것) | 단순화 범위를 정직하게 공개 |

포트폴리오 페이지에는 반드시: 수량·평균단가·현재가(해외는 원통화 병기)·평가액·비중·손익 + **종목 클릭 시 거래내역(lots)** + 현금 행 + 일간변동 + 📒매매일지 + 📅일별현황.

## 4. 데이터 함정 (겪었던 실수 — 재발 방지)

- **yfinance 한국 지수(^KS11/^KQ11)는 하루 지연될 수 있다.** 급락일에 지수만 이전 값에 멈춘 사례 있음 → 글에서 지수 수치를 쓰기 전 `data_date` 확인, 지연이면 **개별 대형주+뉴스로 대체**하고 그 사실을 글에 명시.
- 지수의 `chg_pct`가 **2일 누적**으로 잡힐 수 있다(전일 데이터 지연 시) → 오늘 변동은 어제 종가와 직접 비교.
- **pykrx는 KRX 로그인 요구로 사용 불가** → 한국 유니버스는 FinanceDataReader.
- 스크리너(FDR)는 해외 서버에서 간헐 실패 가능 → 워크플로에서 `continue-on-error`, 실패 시 해당 섹션 생략.
- 클라우드(리눅스) 차트 한글: `fonts-nanum` 설치 필요 (폰트 자동 선택 로직 있음).
- Actions는 실행마다 새 환경 → **data/는 커밋**해야 다음 세션이 참조 가능 (스냅샷 `market-<morning|krclose|uspre>.json`).

## 5. 실행·인증 (요약)

- 분석 모델: `--model opus` (**항상 최신 Opus**). 변경은 `daily.yml`·`run-daily.ps1`.
- 인증: **`CLAUDE_CODE_OAUTH_TOKEN`(구독)만** — Max 사용량 차감. **`ANTHROPIC_API_KEY` 금지**(있으면 API 과금이 우선됨).
- Claude는 **git 명령 금지** — 파일 생성까지만, 커밋·push는 워크플로/스크립트가.

## 6. 글쓰기 원칙

- 근거 없는 단정 금지 — 모든 판단에 데이터/뉴스 근거, 출처 병기.
- 확률로 말한다. 목적은 **적중이 아니라 기록·자기검증** (매 글에 어제 예상 채점 O/△/X + 교훈).
- 데이터 이상(지연·결측)을 발견하면 **숨기지 말고 글에 명시**한다.
- 스크리너 후보는 **뉴스로 재료가 확인된 종목만** 게재, 나머지는 제외 사실을 밝힌다.
- 모든 글 최상단에 "투자 조언 아님" 경고.
