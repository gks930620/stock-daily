# 운영 규칙서 (실수 방지 기준)

> 매일 자동 실행(예상글·매매·게시)이 따라야 하는 확정 규칙. 프롬프트(`prompts/kr-report.md`·`us-report.md`·`portfolio.md`)와
> 코드(`scripts/`)에 이미 반영되어 있으며, 이 문서는 **왜 그렇게 정했는지**와 **재발 방지 체크리스트**의 원본이다.

---

## 0. 역할 분리 — AI 2명 (2026-07-14 확정)

한 회차에 **서로 다른 Claude 세션(=AI 2명)** 이 순서대로 일한다. 헤드리스 `claude -p`를 두 번 호출 → 세션이 분리되어 기억을 공유하지 않는다.

| 순서 | 담당 | 프롬프트 | 하는 일 | 산출물 |
|---|---|---|---|---|
| ① | **애널리스트** | `prompts/<kr\|us>-report.md` | 데이터·차트·뉴스 분석 → **예상글** | `_posts/<날짜>-<kr\|us>-market.md` |
| ② | **포트폴리오 매니저** | `prompts/portfolio.md` | ①의 리포트 + 같은 `market.json` + 현재 보유현황 → **주문 결정** | `portfolio/orders/<날짜>-<kr\|us>.json` |

- 데이터 수집·차트·스크리너·체결계산은 **파이썬**(`scripts/`)이 하고, 두 AI는 그 산출물을 읽고 판단·집필/매매만 한다.
- **손익 비교 가격의 출처는 ① 세션이 만든 게 아니라 파이썬이 수집한 `data/<날짜>/market.json` 하나** — ②의 주문도, 체결·평가(`portfolio.py`)도 모두 이 동일 파일을 기준으로 한다. 그래서 리포트·포트폴리오·손익이 같은 숫자에 묶인다.
- 경계: 애널리스트는 `portfolio/orders`·`_data/portfolio.json`을 건드리지 않는다. 매니저는 `_posts`(글)를 건드리지 않는다. 둘 다 git 금지(커밋은 워크플로).
- 비용: 회차당 xhigh 세션 2개 → 토큰 소모 약 2배. Max 리밋이 부담되면 ② 세션만 `--effort high`로 낮추는 선택지 있음(`daily.yml`).

## 1. 스케줄 (하루 3회, KST)

| 시각 | 모드 | 하는 일 |
|---|---|---|
| **08:00** | `kr` | 수집(morning) + 차트 + 스크리너 + **🇰🇷 한국장 예상글(①)** + 매매결정(②) + 체결 |
| **18:00** | `collect` | 수집만(krclose) — 한국장 마감 결과 스냅샷 + 시가체결·일별평가 확정 (AI 없음) |
| **21:00** | `us` | 수집(uspre) + 차트 + **🇺🇸 미국장 예상글(①)** + 매매결정(②) + 체결 |

- 우리는 **결과 정리가 아니라 "그날 장을 예측"** 한다. 한국글=오늘 9시 개장 예측, 미장글=오늘 밤 개장 예측.
- 검증 루프: 한국글은 어제 한국 예상 vs 실제(krclose), 미장글은 어제 미장 예상 vs 지난밤 실제.

## 2. 글 포맷 — ① 애널리스트 리포트 (결론 → 플레이북 → 체크포인트 → 마무리)

> 2026-07-14 확정: 플레이북형 채택 + **매매 결정은 ② 포트폴리오 매니저로 분리**. 리포트는 "예측"만, 실제 주문표는 넣지 않고 포트폴리오 페이지로 링크. 산문 칼럼 섹션 폐지, **분량 A4 1.5장 이내(번잡함 금지)**.

1. **결론 먼저** — `<div class="verdict">` Q&A 박스 + 한 줄 진단(1~2문장) + 타일(`tiles`) 4~6개
2. **시나리오 플레이북** — `## 🎯 시나리오 플레이북`: 확률 막대(`bars`) + 시나리오 3개 각각 `###` 소제목·불릿(근거/트리거/**무효 조건**/**대응 방침**=전략 방향 수준). 핵심 차트 1~2개만 인라인 — 차트 전용 섹션 없음
3. **체크포인트** — `## 📋 … 체크포인트`: 표(시각 | 확인할 것 | 의미) 3~5행
4. **마무리** — `🔍 비인기 종목 후보`(kr만, **재료 검증된 것만**) → `✅ 어제 채점`(O/△/X + 교훈 + 오늘 시나리오 반영 한 줄) → 맨 끝 한 줄로 `[가상 포트폴리오]` 링크
   - **구체 주문표(수량·금액)는 리포트에 넣지 않는다.** 그건 ② 매니저가 `orders` JSON으로 만들고, 그 `comment`·`reason`이 포트폴리오 페이지의 매매저널로 노출된다.

- 색 규칙(한국식): **상승·강세=빨강**(`up`/`fill-up`), **하락·약세=파랑**(`down`/`fill-down`), 중립=`fill-flat`
- front matter에 `market: kr|us` 필수 (목록 🇰🇷/🇺🇸 배지)
- 파일명: `_posts/YYYY-MM-DD-kr-market.md` / `-us-market.md`

## 3. 가상 포트폴리오 체결 규칙 (1억 페이퍼 트레이딩)

**매매는 하루 단위** (실시간 아님). 주문 결정과 체결 시각이 분리되어 **룩어헤드(결과 보고 사기)가 불가능**하다.

| 규칙 | 내용 | 이유 |
|---|---|---|
| 체결 시각 | **주식·ETF = 그 종목 시장의 "다음 개장 시가"**: 🇰🇷 08시 주문→당일 09:00 시가, 🇰🇷 21시 주문→다음 거래일 시가, 🇺🇸 어느 세션이든→당일 밤 22:30(KST) 개장 시가. 주문은 `pending`으로 대기했다가 시가 확정 후 체결 | 예측 시점엔 체결가를 모름 — 룩어헤드 방지 |
| 24시간 자산 | 암호화폐·원자재는 주문 **즉시** 그 시점 시세 체결 | 항상 열려 있어 "다음 개장" 개념 없음 |
| 평가 | **일별 확정 평가 = 매일 18:00 KST** (한국주=당일 종가·미국주=간밤 종가) → 일별현황·자산곡선의 기준. 페이지 상단 "현재가"는 최근 수집 시점 시세(기준시각 표시) | 실시간 아님을 명시, 매일 같은 시각에 공정하게 채점 |
| 한국 주식 | **정수 주수만** 체결, 잔액은 현금 유지 | 57.8주 같은 비현실 방지 |
| 해외 자산 | 소수점 매수 허용 + **체결 시점 환율(원/달러) 기록** | 해외주식 소수점 거래 관행 |
| 주문서 | `portfolio/orders/<날짜>-<kr|us>.json` — buy는 krw(예산), sell은 qty 생략 시 전량 | 세션별 1회 적용(`applied_orders`) |
| 원칙 | 몰빵 금지·분산·각 주문에 근거(reason)·확신 없으면 hold | 이벤트 직전(발표 전) 추가 베팅 금지 |
| 검증 | 모든 수치는 커밋된 `data/`·`orders/`·`_data/portfolio.json`과 대조 가능 | 신뢰성 — "대충 가상" 금지 |
| 미반영 | 수수료·세금·슬리피지 없음 (명시할 것) | 단순화 범위를 정직하게 공개 |

포트폴리오 페이지에는 반드시: 수량·평균단가·현재가(해외는 원통화 병기)·평가액·비중·손익 + **종목 클릭 시 거래내역(lots)** + 현금 행 + 일간변동 + 📒매매일지 + 📅일별현황.

## 4. 데이터 함정 (겪었던 실수 — 재발 방지)

- **yfinance 한국 지수(^KS11/^KQ11)는 하루 지연될 수 있다.** 급락일에 지수만 이전 값에 멈춘 사례 있음 → 글에서 지수 수치를 쓰기 전 `data_date` 확인, 지연이면 **개별 대형주+뉴스로 대체**하고 그 사실을 글에 명시.
- 지수의 `chg_pct`가 **2일 누적**으로 잡힐 수 있다(전일 데이터 지연 시) → 오늘 변동은 어제 종가와 직접 비교.
- **pykrx는 KRX 로그인 요구로 사용 불가** → 한국 유니버스는 FinanceDataReader.
- 스크리너(FDR)는 해외 서버에서 간헐 실패 가능 → 워크플로에서 `continue-on-error`, 실패 시 해당 섹션 생략.
- 클라우드(리눅스) 차트 한글: `fonts-nanum` 설치 필요 (폰트 자동 선택 로직 있음).
- Actions는 실행마다 새 환경 → **data/는 커밋**해야 다음 세션이 참조 가능 (스냅샷 `market-<morning|krclose|uspre>.json`).

## 5. 실행·인증 (요약)

- 분석 모델: `--model opus` (**항상 최신 Opus**). 변경은 `daily.yml`·`run-daily.ps1`.
- 인증: **`CLAUDE_CODE_OAUTH_TOKEN`(구독)만** — Max 사용량 차감. **`ANTHROPIC_API_KEY` 금지**(있으면 API 과금이 우선됨).
- Claude는 **git 명령 금지** — 파일 생성까지만, 커밋·push는 워크플로/스크립트가.
- 상세: [CLOUD-AUTOMATION.md](CLOUD-AUTOMATION.md)

## 6. 글쓰기 원칙

- 근거 없는 단정 금지 — 모든 판단에 데이터/뉴스 근거, 출처 병기.
- 확률로 말한다. 목적은 **적중이 아니라 기록·자기검증** (매 글에 어제 예상 채점 O/△/X + 교훈).
- 데이터 이상(지연·결측)을 발견하면 **숨기지 말고 글에 명시**한다.
- 스크리너 후보는 **뉴스로 재료가 확인된 종목만** 게재, 나머지는 제외 사실을 밝힌다.
- 모든 글 최상단에 "투자 조언 아님" 경고.
