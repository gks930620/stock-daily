# 매일 주식시장 예상 프로젝트 — 설계 문서

> 📌 이 문서는 **최초 설계(왜 이렇게 만들었나)** 기록입니다. **현재 구현 상태·규칙은 [OVERVIEW.md](OVERVIEW.md)(그림 요약)와 [RULES.md](RULES.md)(운영 규칙서)가 기준**입니다.
> 설계와 달라진 주요 지점: 리포트 목차 → ④결론먼저·①대시보드·②브리핑·③칼럼 포맷(RULES §2), 하루 1회 → 3회(08 🇰🇷/18 수집/21 🇺🇸), 적중률 로그 → 예상글 내 자기검증 + 가상 1억 포트폴리오.
>
> 목표: 매일 아침, 데이터·차트·뉴스를 종합해 **미국·한국 시장의 오늘 시나리오**와
> **오를 가능성이 있는 비인기 종목 후보**를 담은 심층 리포트를 자동 생성한다.
> 최종 판단(결론)은 Claude가 내린다.

---

## 0. 이 프로젝트의 철학 (먼저 합의할 것)

- **"예측 적중"이 아니라 "근거 있는 확률적 시나리오"를 만든다.**
  어떤 AI·애널리스트도 시장을 안정적으로 맞히지 못한다. 우리는 매일
  *상승/하락/보합 시나리오 + 각 확률 + 근거 + 트리거 이벤트*를 기록하고,
  나중에 **적중률을 스스로 검증**하는 것을 목표로 한다.
- **모든 판단에는 근거(데이터/뉴스)를 붙인다.** 근거 없는 "느낌" 금지.
- **투자 조언이 아니다.** 리포트는 리서치·학습 기록이며, 매매는 사용자 책임.
- **재현 가능해야 한다.** 매일 같은 파이프라인 → 같은 형식의 리포트 → 누적 검증.

## 1. 대상 범위

| 구분 | 종목/지수 | 데이터 심볼(예시) |
|------|-----------|-------------------|
| 미국 지수 | S&P500, 나스닥100, 다우 | `^GSPC`, `^NDX`, `^DJI` |
| 미국 리스크지표 | VIX(변동성), 미10년물 금리, 달러인덱스 | `^VIX`, `^TNX`, `DX-Y.NYB` |

> 관심 종목 리스트는 `config/watchlist.yaml`에서 관리(추가/삭제 자유).

## 2. 시스템 구조 (파이프라인)

```
[1] 데이터 수집        [2] 지표 계산·차트     [3] 뉴스·시황       [4] Claude 분석      [5] 저장·검증
   Python                 Python                WebSearch           (본체)              파일 누적
   yfinance/pykrx  ─▶  기술적지표+PNG생성  ─▶  뉴스 요약 텍스트 ─▶  차트+뉴스 종합  ─▶  daily report
   FRED/VIX                                                          → 결론 리포트        + 적중률 리뷰
```

### 데이터 흐름
1. **수집 스크립트**가 가격·거래량·지표·환율을 받아 `data/YYYY-MM-DD/`에 저장(CSV).
2. **차트 스크립트**가 지수·종목별 차트를 PNG로 그려 같은 폴더에 저장.
3. **Claude**가 그 PNG들을 눈으로 읽고 + 웹검색으로 최신 뉴스를 모아서
4. **오늘의 리포트(마크다운)**를 작성 → `_posts/`(현행: `YYYY-MM-DD-{kr,us}-market.md`)에 저장.
5. 다음 날, 전날 예상과 실제 결과를 대조해 **적중률 로그**에 기록.

## 3. 데이터 소스 상세

| 데이터 | 소스 | 비용 | 비고 |
|--------|------|------|------|
| 미국/글로벌 주가·지수 | **yfinance** (Yahoo) | 무료 | 가장 간편. 지연 데이터지만 일봉 분석엔 충분 |
| 한국 주가·지수 | yfinance(`.KS`/`.KQ`) 또는 **pykrx** | 무료 | pykrx는 KRX 공식, 거래량·수급 상세 |
| 경제지표(금리·물가) | **FRED API** | 무료(키) | 미 연준 데이터 |
| 한국 경제지표 | 한국은행 ECOS | 무료(키) | 선택 |
| VIX·공포탐욕 | yfinance / CNN Fear&Greed(웹검색) | 무료 | 센티먼트 |
| 뉴스·시황 | **Claude WebSearch** | 무료 | 실시간 검색 → 요약 |

> **1단계 최소 구성**: `yfinance` + Claude 웹검색만으로 시작. API 키 불필요.
> pykrx·FRED는 심층 분석 강화 시 추가.

## 4. 심층 분석 리포트 구조 (핵심 산출물)

(최초 설계안 — **현행 포맷은 RULES.md §2** 참고. 아래는 초기 목차 기록:)

```
# 오늘의 시장 리포트 — 2026-XX-XX

## 0. 한 줄 결론
   (예: "나스닥 강보합 예상. 반도체 강세 vs 금리 부담. 코스피 관망.")

## 1. 글로벌 매크로 (밤사이 미국장)
   - 3대 지수 마감 / VIX / 금리 / 달러 방향
   - 어젯밤 핵심 이벤트 (실적, 지표 발표, 연준 발언)

## 2. 기술적 분석 (차트 기반)
   - 지수별: 추세, 주요 지지/저항, 이동평균 배열, RSI/MACD 신호
   - 각 차트 이미지에 대한 Claude의 해석

## 3. 뉴스·이벤트 (오늘 주목할 일정)
   - 오늘 예정된 지표 발표/실적/이벤트
   - 밤사이·오전 주요 뉴스 요약 (출처 링크 포함)

## 4. 시장 시나리오 (확률적)
   | 시나리오 | 확률 | 근거 | 트리거 |
   | 상승     | 40%  | ...  | ...    |
   | 보합     | 35%  | ...  | ...    |
   | 하락     | 25%  | ...  | ...    |

## 5. 관심 종목 코멘트
   - watchlist 종목별 오늘의 관전 포인트

## 6. 🔍 오늘의 비인기 종목 후보 (스크리너 결과)
   - 표: 종목 / 발굴 근거(신호) / 리스크 / 주목도
   - Claude의 선별 코멘트 (왜 오를 여지가 있는지 + 왜 위험한지)

## 7. 리스크 & 무효화 조건
   - 이 시나리오가 깨지는 조건 (예: "VIX 25 돌파 시 하락 전환")

## 8. 어제 예상 vs 실제 (자기 검증)
   - 전날 결론이 맞았는지 O/X + 코멘트
```

## 5. 🔍 "오늘 오를 것 같은 비인기 종목" 발굴 로직

> ⚠️ 가장 재미있지만 가장 투기적인 부분. **확정 신호가 아니라 후보 스크리닝**이다.
> Claude는 기계적 신호 + 뉴스 맥락을 결합해 "왜 위험한지"까지 반드시 함께 말한다.

**스크리닝 후보 신호 (Python으로 계산, 조합 사용):**
- **과매도 반등 후보**: RSI < 30 이면서 당일 반등 캔들 출현
- **거래량 급증**: 최근 평균 대비 거래량 2~3배↑ (관심 유입 신호)
- **바닥 다지기**: 장기 하락 후 이동평균 수렴 + 지지선 방어
- **소외 저평가**: 시가총액 소형 + 최근 뉴스 부재였다가 신규 재료 등장
- **52주 신저가 근처 반등** 시도

**Claude의 역할:**
1. 스크리너가 뽑은 후보 목록(수십 개)을 받아
2. 각 후보의 최근 뉴스/재료를 웹검색으로 확인
3. "기계 신호 + 실제 재료"가 맞물리는 소수만 선별
4. 각 종목에 **상승 근거 / 리스크 / 신뢰도** 를 명시

> ⚠️ 소형·비인기주는 변동성·유동성 리스크가 크다. 리포트에 항상 경고 문구 포함.

## 6. 자동화 (선택, 나중 단계)

- **스케줄**: 매일 장 시작 전(예: 한국시간 08:00) 자동 실행
  - Claude Code의 스케줄 기능 또는 OS 작업 스케줄러 사용
- **흐름**: 데이터수집 → 차트생성 → Claude 리포트작성 → 파일저장 → (선택) 알림
- 1단계에서는 **수동 실행**으로 검증 후, 안정되면 자동화 전환.

## 7. 폴더 구조 (실제 구현)

```
주식시장예상클로드코드/
├── docs/                       ← 설계·규칙 문서 (RULES.md가 운영 기준)
├── config/watchlist.yaml       ← 수집 종목 설정 (~45개)
├── prompts/{kr,us}-report.md   ← 세션별 예상글 지시문
├── scripts/
│   ├── collect_data.py         ← 시세+FRED 수집 (+시점 스냅샷)
│   ├── make_charts.py          ← 차트 PNG
│   ├── screener.py             ← 비인기 종목 스크리너 (FinanceDataReader)
│   ├── portfolio.py            ← 가상 1억 체결·평가·자산곡선
│   └── run-daily.ps1           ← 로컬 실행 (백업)
├── data/YYYY-MM-DD/            ← 시세 JSON (morning/krclose/uspre, 검증용 커밋)
├── _posts/                     ← 예상글 (YYYY-MM-DD-{kr,us}-market.md)
├── portfolio/orders/           ← 세션별 매매 주문서
├── _data/portfolio.json        ← 포트폴리오 상태
├── assets/{charts,portfolio}/  ← 차트·자산곡선 이미지
└── .github/workflows/daily.yml ← 하루 3회 자동 실행
```

## 8. 개발 단계 (로드맵)

- ✅ **1단계 (MVP)**: yfinance 수집 + 차트 + 샘플 리포트 — 완료
- ✅ **2단계**: 포맷 확정(④①②③) + 스크리너(FDR) + FRED 경제지표 — 완료
- ✅ **3단계**: 예상글 내 자기검증(✅어제 vs 실제) + **가상 1억 포트폴리오**(체결·일지·자산곡선) — 완료
- ✅ **4단계**: GitHub Actions 하루 3회 자동 실행(08/18/21 KST, Max 구독 인증) — 완료
- ⏳ **이후 (선택)**: 알림(메일/디스코드), 적중률 통계 페이지, 종목 확장 등

---

## 9. 한계와 주의 (반드시 인지)

- 시세는 **지연 데이터**일 수 있음(장중 실시간 아님). 일봉·시황 분석엔 충분.
- **미래 예측의 신뢰성은 근본적으로 제한적.** 적중률로 겸손하게 검증한다.
- 비인기·소형주 후보는 **투기적**이며 유동성 리스크가 큼.
- 본 리포트는 **투자 조언이 아니다.** 최종 매매 판단·책임은 사용자에게 있다.
